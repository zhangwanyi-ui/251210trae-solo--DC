# DC在线业务系统页面构建提示词

## 项目概述
基于React 18、Vite 5、React Router 6和Ant Design 5，构建一个名为"DC"的在线业务系统页面，严格按照提供的页面结构文档实现。

## 技术栈要求
- React 18
- Vite 5
- React Router 6
- Ant Design 5
- Redux Toolkit（可选，用于状态管理）
- Axios（可选，用于API请求）
- Less（用于样式）

## 页面结构设计
### 1. 整体布局
- 采用三栏布局：顶部导航栏 + 左侧导航栏 + 内容区域
- 顶部导航栏：显示系统名称和功能入口导航（报警、事件、任务、在线、个人中心）
- 左侧导航栏：显示左侧导航页（全景俯瞰、在线业务、应用服务、基础资源、服务健康）
- 内容区域：显示页面名称、面包屑导航、内容导航项和页面内容

### 2. 导航系统设计
#### 顶部导航（功能入口导航）
- 包含：报警、事件、任务、在线、个人中心
- 每个导航项包含下拉菜单，显示具体功能页面
- 点击顶部导航进入的页面，不显示内容导航项，但保留面包屑导航
- 页面路径示例：/alarm/alarm-status, /event/business-operation-events

#### 左侧导航
- 包含5个一级菜单，17个二级菜单：
  - 全景俯瞰：运营全景、全过程价值链、资源池
  - 在线业务：访客、渠道、行为、内容
  - 应用服务：流量承载、在线体验(性能体验)、服务传递效率(和瓶颈)
  - 基础资源：存量、服务可用性、使用和转化(绩效)
  - 服务健康：健康状态（体检）报告（周期性检测报告）、黑名单状态、常见问题、健康干预绩效
- 点击左侧导航进入的页面，显示内容导航项
- 页面路径示例：/overview/operation-overview, /online-business/visitors

#### 内容导航
- 仅在左侧导航页面显示，顶部导航页面不显示
- 采用二级级联选择器实现，一级为分类，二级为具体内容：
  - 观察：实时动态、近期发生、历史回顾
  - 感知：变化分析报告、对比分析报告、分布特征分析报告、归因分析报告、相关性分析报告
  - 关注：我的个人关注、推荐的关注、强制关注
  - 行动：处理任务变更操作、响应操作、参与协同操作
- 选中内容导航项后，面包屑导航应跟随更新

### 3. 页面元素要求
- **页面名称**：显示在面包屑导航上方，字体大小20px，字体粗细600
- **面包屑导航**：显示当前页面路径，仅在有对应页面的地方可点击
  - 示例：首页 / 全景俯瞰 / 运营全景 / 观察 / 实时动态
  - 仅末级导航项可点击跳转
- **内容导航**：二级级联选择器，宽度300px，支持搜索
- **页面内容区域**：白色背景，阴影效果，包含占位文本

## 代码实现要求
### 1. 项目结构
```
src/
├── components/
│   ├── ContentNav.jsx      # 内容导航组件
│   ├── SideNav.jsx        # 左侧导航组件
│   └── TopNav.jsx         # 顶部导航组件
├── pages/
│   ├── BasePage.jsx       # 基础页面组件
│   └── ...                # 其他页面组件
├── router/
│   └── index.jsx          # 路由配置
├── styles/
│   ├── index.less         # 全局样式
│   └── theme.less         # 主题样式
├── App.jsx                # 根组件
└── main.jsx               # 入口文件
```

### 2. 关键组件实现
#### ContentNav.jsx
- 实现面包屑导航和内容导航项
- 顶部导航页面不显示内容导航项
- 内容导航使用Ant Design Cascader组件
- 面包屑导航根据当前路径和内容导航选中值动态生成

#### SideNav.jsx
- 实现左侧导航菜单，支持折叠
- 采用互斥原则，只有一个导航项可以展开
- 使用Ant Design Menu组件，mode="inline"

#### TopNav.jsx
- 实现顶部导航菜单，包含系统名称和功能入口
- 下拉菜单使用Ant Design Menu组件
- 使用Ant Design Header组件

#### BasePage.jsx
- 作为所有页面的基础组件，显示页面名称、ContentNav组件和页面内容
- 接收title属性，用于显示页面名称

### 3. 路由配置
- 使用React Router 6的createBrowserRouter配置所有路由
- 配置272个页面的路由：17个左侧导航页 + 17*14个内容导航页 + 17个功能入口导航页
- 所有页面均使用BasePage组件渲染

## 构建和运行说明
1. 创建Vite项目：`npm create vite@latest . -- --template react`
2. 安装依赖：`npm install antd @ant-design/icons react-router-dom @reduxjs/toolkit axios less`
3. 配置Vite：在vite.config.js中添加Less支持和路径别名
4. 编写代码：按照上述要求实现所有组件和页面
5. 构建项目：`npm run build`
6. 运行项目：`npm run dev`
7. 访问地址：http://localhost:3000

## 页面验证要求
1. 顶部导航进入的页面：显示页面名称、面包屑导航，不显示内容导航项
2. 左侧导航进入的页面：显示页面名称、面包屑导航、内容导航项
3. 内容导航项选中后，面包屑导航应跟随更新
4. 面包屑导航仅在有对应页面的地方可点击
5. 左侧导航展开收起遵循互斥原则
6. 页面名称显示在面包屑导航上方
7. 所有页面均可正常访问，无404错误

## 注意事项
1. 严格按照提供的页面结构文档实现，不得随意增减页面或导航项
