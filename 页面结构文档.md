# DC在线业务系统 - 页面结构文档

## 1. 文档概述

### 1.1 文档目的

本文档详细描述了DC在线业务系统的页面结构，包括页面总数、导航体系、页面模板、页面映射关系等，为前端开发团队提供明确的页面设计和开发依据，确保系统页面结构清晰、导航合理、用户体验良好。

### 1.2 术语定义

| 术语 | 解释 |
|------|------|
| 左侧导航页 | 位于页面左侧的分类导航，包含5个分类的17个导航页 |
| 内容导航项 | 位于内容区域的导航，包含14个内容导航项，用于动态切换页面内容 |
| 功能入口导航项 | 位于顶部导航区的功能入口，包含5个功能入口的17个页面 |
| 页面模板 | 14个内容导航项对应的页面结构，用于快速构建272个页面 |

## 2. 页面总数计算

系统总页面数量为272页，具体计算方式如下：

```
总页面数 = 左侧导航页（17页） + 左侧导航×内容导航（17×14=238页） + 功能入口导航（17页）
        = 17 + 238 + 17
        = 272页
```

## 3. 导航体系设计

### 3.1 左侧导航（17页）

左侧导航包含5个分类，每个分类下包含若干导航页，总计17个左侧导航页：

| 分类 | 导航页 | 数量 |
|------|--------|------|
| 全景俯瞰 | 运营全景、全过程价值链、资源池 | 3 |
| 在线业务 | 访客、渠道、行为、内容 | 4 |
| 应用服务 | 流量承载、在线体验(性能体验)、服务传递效率(和瓶颈) | 3 |
| 基础资源 | 存量、服务可用性、使用和转化(绩效) | 3 |
| 服务健康 | 健康状态（体检）报告（周期性检测报告）、黑名单状态、常见问题、健康干预绩效 | 4 |
| **总计** | - | **17** |

### 3.2 内容导航（14项）

每个左侧导航页对应14个内容导航项，总计17×14=238个内容页面。内容导航项按功能分为4个类别：

| 类别 | 内容导航项 | 数量 | 页面模板类型 |
|------|------------|------|--------------|
| 观察 | 实时动态、近期发生、历史回顾 | 3 | 监控页、列表页、列表页 |
| 感知 | 变化分析报告、对比分析报告、分布特征分析报告、归因分析报告、相关性分析报告 | 5 | 报表页 |
| 关注 | 我的个人关注、推荐的关注、强制关注 | 3 | 列表页 |
| 行动 | 处理任务变更操作、响应操作、参与协同操作 | 3 | 表单页 |
| **总计** | - | **14** | - |

### 3.3 功能入口导航（17页）

位于顶部导航区，包含5个功能入口，每个功能入口包含若干子页面，总计17个功能入口页面：

| 功能入口 | 子页面 | 数量 |
|----------|--------|------|
| 报警 | 报警状态、督促、报警处理绩效 | 3 |
| 事件 | 业务运行事件、岗位事件、任务事件、绩效事件 | 4 |
| 任务 | 应急响应任务、视情干预计划任务、常态化任务 | 3 |
| 在线 | 岗位责任情况报告、在线值守报告、岗位责任涉及报告、岗位综合绩效报告 | 4 |
| 个人中心 | 注册信息、个性化设置、我的内容 | 3 |
| **总计** | - | **17** |

## 5. 页面URL设计规范

为了保证系统页面URL的规范性和可维护性，建议采用以下URL设计规范：

### 5.1 左侧导航页URL

```
/[左侧导航分类]/[左侧导航页名称]
```

**示例：**
- 全景俯瞰-运营全景：`/overview/operation-overview`
- 在线业务-访客：`/online-business/visitors`
- 应用服务-流量承载：`/application-service/traffic-bearing`

### 5.2 内容导航页URL

```
/[左侧导航分类]/[左侧导航页名称]/[内容导航项名称]
```

**示例：**
- 全景俯瞰-运营全景-实时动态：`/overview/operation-overview/realtime-dynamic`
- 在线业务-访客-变化分析报告：`/online-business/visitors/change-analysis`
- 服务健康-常见问题-处理任务变更操作：`/service-health/common-issues/task-change`

### 5.3 功能入口导航页URL

```
/[功能入口名称]/[子页面名称]
```

**示例：**
- 报警-报警状态：`/alarm/alarm-status`
- 事件-业务运行事件：`/event/business-operation-event`
- 个人中心-注册信息：`/personal/registration-info`

## 5. 页面模板设计详解

### 5.1 监控页模板（实时动态）

#### 5.1.1 模板概述
用于展示实时数据动态，支持数据实时更新和预警提示。

#### 5.1.2 布局结构
```
┌─────────────────────────────────────────────────────────────────┐
│  页面标题 + 面包屑导航                                           │
├─────────────────────────────────────────────────────────────────┤
│  筛选区：时间范围选择器、数据维度选择器、筛选条件表单             │
├─────────────────────────────────────────────────────────────────┤
│  实时数据概览区：关键指标卡片，展示核心数据和状态                 │
├─────────────────────────────────────────────────────────────────┤
│  图表展示区：多种图表类型（折线图、柱状图、饼图等），数据实时更新 │
├─────────────────────────────────────────────────────────────────┤
│  数据表格区：实时数据列表，支持自动刷新和手动刷新                 │
├─────────────────────────────────────────────────────────────────┤
│  操作区：导出数据、设置预警阈值等功能按钮                         │
└─────────────────────────────────────────────────────────────────┘
```

#### 5.1.3 核心组件
- 时间范围选择器：支持实时、最近1小时、最近24小时、自定义时间范围
- 数据维度选择器：支持按设备、区域、业务等维度筛选
- 关键指标卡片：展示核心指标，支持状态色标识（正常/警告/错误）
- ECharts图表：支持多种图表类型，实时更新数据
- 数据表格：支持自动刷新（可配置刷新间隔），支持排序、筛选

#### 5.1.4 交互设计
- 数据自动刷新：默认30秒刷新一次，可配置刷新间隔
- 图表交互：支持缩放、拖拽、数据点提示
- 预警提示：指标超过阈值时，卡片和图表显示警告或错误状态

### 5.2 列表页模板（近期发生、历史回顾、我的个人关注、推荐的关注、强制关注）

#### 5.2.1 模板概述
用于展示列表数据，支持筛选、排序、分页等功能。

#### 5.2.2 布局结构
```
┌─────────────────────────────────────────────────────────────────┐
│  页面标题 + 面包屑导航                                           │
├─────────────────────────────────────────────────────────────────┤
│  操作区：新增、导入、导出、搜索、筛选等按钮                       │
├─────────────────────────────────────────────────────────────────┤
│  数据列表：支持多选、排序、筛选、分页                             │
├─────────────────────────────────────────────────────────────────┤
│  分页控件：显示总条数、当前页码、每页条数选择                     │
└─────────────────────────────────────────────────────────────────┘
```

#### 5.2.3 核心组件
- 搜索框：支持关键词搜索
- 筛选表单：支持多条件组合筛选
- 数据表格：支持多选、排序、行操作
- 分页控件：支持自定义每页条数

#### 5.2.4 交互设计
- 行悬停效果：高亮显示当前行
- 行操作菜单：支持查看、编辑、删除等操作
- 批量操作：支持多选后批量处理
- 分页交互：支持页码跳转、上一页/下一页

### 5.3 报表页模板（变化分析报告、对比分析报告、分布特征分析报告、归因分析报告、相关性分析报告）

#### 5.3.1 模板概述
用于生成和展示各种分析报告，支持数据可视化和导出。

### 5.4 表单页模板（处理任务变更操作、响应操作、参与协同操作）

```
└─────────────────────────────────────────────────────────────────┘
```

#### 5.4.3 核心组件
- 表单控件：输入框、下拉选择、日期选择、单选、多选等

- 保存草稿：支持自动保存和手动保存草稿
- 提交反馈：提交成功/失败显示对应提示

## 7. 页面开发规范

### 7.1 组件命名规范

- **组件名称**：使用大驼峰命名法，如 `TopNav`、`SideNav`
- **文件名称**：使用大驼峰命名法，与组件名称保持一致，如 `TopNav.jsx`、`TopNav.css`
- **CSS类名**：使用BEM命名规范，如 `top-nav__logo`、`side-nav__menu-item--active`

### 7.2 代码结构规范

```
src/
├── components/          # 通用组件
│   ├── TopNav/         # 顶部导航组件
│   │   ├── TopNav.jsx  # 组件代码
│   │   ├── TopNav.css  # 组件样式
│   │   └── index.js    # 组件导出
│   ├── SideNav/        # 左侧导航组件
│   └── ContentArea/    # 内容区域组件
├── pages/              # 页面组件
│   ├── DefaultPage/    # 默认页面模板
│   └── ...             # 其他页面
├── routes/             # 路由配置
│   └── routes.jsx      # 路由定义
├── services/           # API服务
│   ├── api.js          # API封装
│   └── mockData.js     # Mock数据
├── store/              # 状态管理
│   ├── slices/         # Redux slices
│   └── index.js        # Store配置
├── utils/              # 工具函数
│   ├── format.js       # 数据格式化
│   └── validate.js     # 数据验证
├── styles/             # 全局样式
│   ├── variables.less  # 样式变量
│   └── global.css      # 全局样式
└── App.jsx             # 应用入口组件
```

### 7.3 路由配置规范

- 使用React Router 6的声明式路由
- 实现路由懒加载，提高初始加载速度
- 配置404页面
- 实现路由守卫，进行权限控制

### 7.4 状态管理规范

- 使用Redux Toolkit进行状态管理
- 按功能模块划分slices
- 实现异步数据处理
- 合理使用selectors获取状态

### 7.5 性能优化规范

- 实现组件懒加载：使用React.lazy和Suspense
- 实现路由懒加载：按路由分割代码
- 使用React.memo和useMemo优化组件渲染
- 实现数据缓存：避免重复请求
- 优化图片加载：使用适当尺寸和格式的图片
- 实现虚拟列表：处理大量数据列表

### 7.6 可访问性规范

- 使用语义化HTML标签
- 为图片添加alt属性
- 确保表单控件有正确的label关联
- 支持键盘导航
- 实现适当的颜色对比度
- 支持屏幕阅读器

### 7.7 代码质量规范

- 使用ESLint进行代码检查
- 使用Prettier进行代码格式化
- 编写单元测试：覆盖率≥80%
- 编写清晰的注释
- 遵循一致的代码风格

### 7.8 版本控制规范

- 遵循Git Flow工作流
- 提交信息使用规范格式：`type(scope): description`
- 定期合并代码到主分支
- 实现CI/CD流水线

## 8. 页面开发建议

1. **组件化开发**：将页面拆分为可复用的组件，如导航组件、表格组件、图表组件等，提高开发效率和代码可维护性。

2. **路由设计**：使用React Router 6进行路由管理，实现页面的动态加载和切换。

3. **状态管理**：使用Redux Toolkit进行状态管理，统一管理页面状态和数据。

4. **响应式设计**：确保页面在不同屏幕尺寸下都能正常显示和使用。

5. **性能优化**：实现页面懒加载、组件懒加载、数据缓存等性能优化措施，提高页面加载速度和响应时间。

6. **SEO优化**：对页面标题、meta标签、URL等进行SEO优化，提高页面搜索引擎排名。

7. **可访问性**：确保页面符合WCAG可访问性标准，支持键盘导航和屏幕阅读器。

8. **测试驱动开发**：编写测试用例，确保代码质量和功能正确性。

9. **文档化**：编写组件文档和API文档，提高代码可维护性。

10. **持续集成**：实现CI/CD流水线，确保代码质量和快速部署。
