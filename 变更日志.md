# DC在线业务系统 - 变更日志

## 1. 文档概述

### 1.1 文档目的
本变更日志旨在记录DC在线业务系统的版本变更历史，包括功能新增、bug修复、性能优化和其他重要变更，帮助开发人员、测试人员和用户了解系统的演进过程。

### 1.2 版本号规则
采用语义化版本号（Semantic Versioning）：
- **主版本号**：当进行不兼容的API变更时递增
- **次版本号**：当添加向下兼容的新功能时递增
- **修订号**：当进行向下兼容的bug修复时递增

格式：`v{主版本号}.{次版本号}.{修订号}`，例如：`v1.0.0`

### 1.3 变更类型
- **新增**：添加新功能或特性
- **修复**：修复bug或问题
- **优化**：优化性能、用户体验或代码结构
- **变更**：修改现有功能或API
- **移除**：移除功能或API
- **安全**：安全相关的修复或增强

## 4. 升级指南

### 4.1 从v1.2.x升级到v1.3.x

#### 数据库迁移
1. 执行数据库迁移脚本：
   ```bash
   npm run migrate
   ```

#### 配置变更
1. 新增AI配置项：
   ```
   AI_API_KEY=your_ai_api_key
   AI_MODEL=your_ai_model
   ```

2. 新增大数据配置项：
   ```
   BIGDATA_ENABLED=true
   BIGDATA_CLUSTER=your_cluster_address
   ```

#### 代码修改
1. 更新依赖包：
   ```bash
   npm update
   ```

2. 修改部分API调用，适配新的接口格式

### 4.2 从v1.3.x升级到v1.4.x

#### 数据库迁移
1. 执行数据库迁移脚本：
   ```bash
   npm run migrate
   ```

#### 配置变更
1. 新增AR/VR配置项：
   ```
   ARVR_ENABLED=true
   ARVR_MODEL_PATH=your_model_path
   ```

2. 新增边缘计算配置项：
   ```
   EDGE_COMPUTING_ENABLED=true
   EDGE_NODES=your_edge_nodes
   ```

#### 代码修改
1. 更新依赖包：
   ```bash
   npm update
   ```

2. 修改部分组件，适配新的功能
